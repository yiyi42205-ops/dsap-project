* **題目名稱**：高效能本機文件搜尋引擎 (C++ High-Performance Local Search Engine)
* **動機與目標**：
    在處理大量純文字或 Markdown 筆記時，系統內建搜尋往往速度緩慢且不支援進階排序。本專案目標是利用 C++ 開發一個能針對特定資料夾進行快速索引與全文檢索的工具，並提供相關度排序功能。
* **預期功能**：
    1.  **資料夾掃描**：自動讀取指定路徑下的所有 `.txt` 與 `.md` 檔案。
    2.  **全文檢索**：輸入關鍵字，瞬間找出包含該字的所有文件。
    3.  **自動補全 (Autocomplete)**：輸入部分字根時，顯示可能的完整單字。
    4.  **相關度排序**：根據關鍵字出現頻率 (TF-IDF 概念) 進行結果排序。
* **使用的技術 / 語言**：
    * 語言：C++ 17 或以上。
    * 標準庫：`std::filesystem` (檔案管理)、`std::unordered_map` (雜湊表)、`std::vector` (動態陣列)。
* **初步時程規劃**：
    * Week 7 - 8：實作檔案讀取模組與基礎 Tokenizer (分詞器)。
    * Week 9 - 10：實作 Trie 與 Inverted Index 資料結構。
    * Week 11：完成 Prototype 並進行初步測試。
    * Week 12 - 14：實作排序演算法與多執行緒優化。
    * Week 15：完成最終測試與錄製 Demo 影片。

#### **專題涉及的資料結構或演算法**
1.  **Trie (字典樹)**：用於儲存所有出現過的單字，支援高效的關鍵字「自動補全」功能。
2.  **Inverted Index (倒排索引)**：這是搜尋引擎的核心。使用 `unordered_map<string, vector<DocID>>` 結構，讓搜尋時間複雜度趨近於 $O(1)$ 或 $O(L)$ (L 為關鍵字長度)。
3.  **Sorting Algorithm**：應用 `std::sort` 並自定義比較函數，根據詞頻對搜尋結果進行排序。
4.  **File I/O 與字串處理**：涉及大量字串拆解與過濾非必要符號的邏輯。
